# Introduction

TradingHunter is an AI agent trading sandbox. You build agents that trade cryptocurrency pairs using programmable strategies, then deploy them into a paper-trading arena where they compete against other agents — all without risking real capital.

The platform provides real-time market data, backtesting infrastructure, a strategy SDK, and a genetic evolution engine. Your agents run 24/7 in isolated sandbox environments.

TIP
TradingHunter is a paper-trading sandbox. No real money is at risk. All orders, balances, and P&L are simulated. This is by design — the sandbox exists so you can iterate fast and fail safely.

Key features

  • Agent SDK — Build trading agents in JavaScript with full control over signal processing, entry/exit logic, and risk parameters
  • Backtesting — Run strategies against years of historical data with realistic slippage and fee models
  • Arena Mode — Deploy agents to compete head-to-head against other traders' bots in live sandbox markets
  • Evolution Lab — Use genetic algorithms to breed and mutate agent strategies that no human would design
  • Real-time terminal — Monitor agents, signals, and positions through a live terminal interface

# Quickstart

Create your first trading agent in under five minutes.

javascript
import { Agent, Strategy } from '@tradinghunter/sdk'; // Define a simple momentum strategy class MomentumReversal extends Strategy { configure() { return { pair: 'BTC/USD', timeframe: '5m', ema: { fast: 9, slow: 21 }, rsi: { period: 14, oversold: 30, overbought: 70 }, risk: { maxDrawdown: 0.08, positionSize: 0.025 } }; } evaluate(candles, indicators) { const { ema, rsi } = indicators; if (ema.fast > ema.slow && rsi < 35) { return { action: 'LONG', confidence: 0.82 }; } if (ema.fast < ema.slow && rsi > 65) { return { action: 'SHORT', confidence: 0.74 }; } return { action: 'HOLD', confidence: 0 }; } } // Create and deploy your agent const agent = new Agent({ name: 'momentum_hunter_01', strategy: new MomentumReversal(), capital: 10_000, sandbox: true }); await agent.deploy(); // → Agent #0047 LIVE in sandbox
TIP
The sandbox: true flag ensures your agent trades with simulated capital. Never set this to false unless you understand the implications — the sandbox is where iteration happens.

# Strategies

Strategies are the core logic of your agent. They receive market data and indicators, then return trade signals. TradingHunter ships with four built-in strategies and a framework for custom ones.

Built-in strategies

momentum — Trend-following with EMA crossover ▶

Generates LONG signals when the fast EMA crosses above the slow EMA and RSI is below oversold. Generates SHORT signals on the inverse. Best for trending markets with clear direction.

ema.fastnumberFast EMA period (default: 9)
ema.slownumberSlow EMA period (default: 21)
rsi.periodnumberRSI calculation period (default: 14)
mean_rev — Mean reversion with Bollinger Bands ▶

Identifies overbought/oversold conditions using Bollinger Bands and RSI. Enters LONG when price touches the lower band with RSI oversold confirmation. Best for ranging markets.

bb.periodnumberBollinger Band period (default: 20)
bb.stdDevnumberStandard deviation multiplier (default: 2.0)
rsi.periodnumberRSI period (default: 14)
breakout — Volatility breakout with ATR ▶

Uses Average True Range (ATR) to detect volatility expansion. Enters when price breaks above/below the ATR channel. Best for high-volatility regimes.

atr.periodnumberATR period (default: 14)
atr.multipliernumberATR multiplier for channel width (default: 1.5)
custom — Write your own strategy class ▶

Extend the Strategy base class and implement configure() and evaluate(). You have full access to candle data, all built-in indicators, and custom indicator functions.

See Custom Indicators for advanced usage.

Strategy interface

typescript

# Risk Management

Every agent has built-in risk controls. These are enforced at the execution layer — your strategy can signal a trade, but the risk engine decides whether it's allowed.

WARNING
Setting maxDrawdown to 1.0 disables the drawdown circuit breaker. Your agent can lose its entire simulated capital. Only use this in backtesting — never in arena mode.

Risk parameters

ParameterTypeDefaultDescription
maxDrawdownnumber0.08Maximum portfolio drawdown (0–1). Agent stops if exceeded.
positionSizenumber0.025Fraction of capital per trade (0–1).
stopLossnumber0.032Default stop-loss as fraction of entry price.
takeProfitnumber0.068Default take-profit as fraction of entry price.
maxPositionsnumber5Maximum simultaneous open positions.
cooldownMsnumber30000Minimum time between trades (ms).

Drawdown circuit breaker

When the agent's portfolio drawdown exceeds maxDrawdown, the risk engine:

  1. Closes all open positions at market price
  2. Sets agent state to STOPPED
  3. Emits a drawdown event
  4. If autoRestart is true, waits 60 seconds then restarts with reduced position sizing

# Backtesting

Before deploying to the arena, test your strategy against historical data. Backtesting simulates realistic conditions including slippage, fees, and latency.

javascript
const result = await agent.backtest({ pair: 'BTC/USD', period: '365d', capital: 10_000, fees: { maker: 0.001, taker: 0.0015 }, slippage: 0.0005, // 0.05% slippage model latency: 200 // 200ms simulated latency }); // Results console.log(result.summary); // { // roi: +23.7%, // maxDrawdown: 6.2%, // sharpe: 1.84, // trades: 142, // winRate: 64%, // avgHoldTime: '4h 23m' // }

Backtest metrics

MetricDescription
roiTotal return on investment including fees
maxDrawdownMaximum peak-to-trough decline
sharpeRisk-adjusted return (Sharpe ratio)
tradesTotal number of executed trades
winRatePercentage of profitable trades
avgHoldTimeAverage position duration

# Arena Mode

The arena is where agents compete. Your agent trades against other traders' bots in live sandbox markets with real-time price feeds. Positions, P&L, and rankings are tracked on a public leaderboard.

javascript
// Deploy to arena const arenaId = await agent.deploy({ arena: 47, visibility: 'public' // 'public' | 'private' }); // Listen for arena events agent.on('trade', (trade) => { console.log(`Trade: ${trade.side} ${trade.size} ${trade.pair} @ ${trade.price}`); }); agent.on('drawdown', (event) => { console.log(`Drawdown alert: ${event.currentDrawdown}%`); });
TIP
Arena matches run for 7 days. After each match, agents are ranked by risk-adjusted return (Sharpe ratio × ROI). The top 3 agents receive permanent leaderboard placement.

# Evolution Lab new

The Evolution Lab uses genetic algorithms to breed and mutate agent strategies. You provide a base strategy and fitness criteria; the engine generates variants, backtests them, and selects the fittest for the next generation.

javascript
import { EvolutionLab } from '@tradinghunter/sdk'; const lab = new EvolutionLab({ baseStrategy: MomentumReversal, populationSize: 50, generations: 20, mutationRate: 0.15, crossoverRate: 0.7, fitness: 'sharpe', // Optimize for Sharpe ratio backtestPeriod: '180d', eliteCount: 5 // Preserve top 5 each generation }); const result = await lab.evolve(); console.log(result.best); // { // generation: 17, // fitness: 2.41, // params: { ema.fast: 7, ema.slow: 23, rsi.period: 12, ... } // }

# Custom Indicators

Write your own indicator functions and use them inside any strategy.

javascript
import { Indicator } from '@tradinghunter/sdk'; class VWAP extends Indicator { compute(candles) { let cumulativeVolume = 0; let cumulativeTP = 0; return candles.map(c => { const typicalPrice = (c.high + c.low + c.close) / 3; cumulativeVolume += c.volume; cumulativeTP += typicalPrice * c.volume; return cumulativeTP / cumulativeVolume; }); } } // Register and use agent.useIndicator('vwap', new VWAP()); // Access in strategy.evaluate() const vwap = indicators.vwap;

# FAQ

Is TradingHunter real trading? ▶

No. TradingHunter is a paper-trading sandbox. All capital, orders, and P&L are simulated. No real money is ever at risk. The platform is designed for strategy development, backtesting, and competitive AI agent research.

What programming languages are supported? ▶

The SDK is available for JavaScript/TypeScript (Node.js 18+). Python and Rust SDKs are on the roadmap. The REST API and webhooks can be used from any language.

How often do agents trade? ▶

Agents evaluate signals on every new candle for their configured timeframe. A 5m strategy checks for signals every 5 minutes. You can set a cooldownMs parameter to enforce minimum time between trades (default: 30 seconds).

Can I run multiple agents simultaneously? ▶

Yes. The free plan supports 3 concurrent agents. Pro supports 25. Enterprise is unlimited. Each agent gets its own isolated portfolio and risk engine.

What happens when an agent hits max drawdown? ▶

The risk engine closes all open positions, sets the agent status to STOPPED, and emits a drawdown event. If autoRestart is enabled, the agent restarts after 60 seconds with reduced position sizing. You can also restart manually via the CLI or API.

How do arena matches work? ▶

Arena matches run for 7 days. All agents trade the same simulated market with identical price feeds. Rankings are based on risk-adjusted return (Sharpe ratio × ROI). The top 3 agents receive permanent leaderboard placement. After a match, you can review your agent's trades, signals, and performance in the terminal.
